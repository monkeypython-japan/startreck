# Startreck 仕様書レビュー

**対象ファイル:** startreck-sprec.md  
**レビュー観点:** Python 実装を想定した一貫性・定義の充足性

---

## 1. 表記の誤り・単位の不統一  **<= 済**

| 箇所 | 問題 |
|------|------|
| 基地の耐久性 | `20000 gl` → `gj` の誤り |
| ミサイルの起爆距離 | `<10 grit>` → `grid` の誤り |
| 特務艦ミサイル威力 | `威力: 500` → 単位 `gj` が欠落 |
| BOTコマンダー | `補給時点` → `補給地点` の誤り |

---

## 2. 定義の矛盾

### ビームのダメージモデルが2箇所で食い違っている  **<= 済**

- **兵装セクション:** 「移動中に接触したビーム・恒星以外のオブジェクトにダメージを与える」(通過型)
- **ビーム詳細セクション:** 「先端から10 grid以内は100%、20 gridまでは25%のダメージ」(爆発範囲型)

どちらが正しいか、あるいは両方を組み合わせるのか不明。

### 補給の発動条件が不統一  **<= 済**

- 基地: 「50 grid 以内」と明記
- 艦艇間補給 (重巡洋艦): 補給範囲の距離が未定義
- 艦艇の補給に関する記述: 「接触」とあるが定量的でない

### UIの速度指定が混在  **<= 済**

- メニューの選択肢: `(100%, 75%, 50%, 25%, 10 grid/sec, 5 grid/sec, 1 grid/sec)`
- パーセントと絶対値が混在。何の100%なのか(最大速度?)も不明

---

## 3. 大きく不足している定義

### 移動のエネルギー消費式がない **<= 済**

「船の移動でエネルギーを消費する」とあるが、速度・距離に対する消費量の計算式が一切定義されていない。実装不能。

### ジェネレータ劣化の計算式がない **<= 削除**

「ダメージが蓄積するにつれてジェネレータの発生率が劣化する」とあるが、劣化カーブ(線形? 段階的?)が未定義。

### BOTコマンダーの補給判断閾値がない **<= 済**

「自艦のダメージが大きい場合は補給地点を目指す」とあるが、「大きい」の基準(例: 耐久性の50%以下)が未定義。

### 衝突判定の半径・サイズがない  **<= 済**

すべてのオブジェクトに「大きさ」が定義されていない(恒星だけ `大きさ(grid)` がある)。衝突検出のための物体サイズが不明。

### 勝利条件のチェック主体がない  **<= 済**

宇宙(Universe)オブジェクトの「振る舞い」にゲーム終了判定が含まれていない。

### 初期状態が未定義　**<= 済**

- 各艦の燃料・ミサイルの初期搭載量(満載? 仕様の何%?)
- キャパシタの初期チャージ量
- 基地周辺への駆逐艦の具体的な配置半径

---

## 4. 設計上の曖昧さ

### 連邦駆逐艦のインテグレータ共有範囲　**<= 済**

BOTコマンダーは「インテグレータの情報を元に行動する」が、各艦が独立したインテグレータを持つのか、艦隊で共有するのかが不明。Python のオブジェクト設計に直結する。

### ミサイルの所属識別　**<= 済**

ミサイルは発射後に独立したオブジェクトになるが、「どの陣営のミサイルか」を保持する属性が仕様に明示されていない(UIでの色分けには必要)。

### ビームが自陣にダメージを与えるか　**<= 済**

「自身以外の艦艇」とあるが、発射した自艦の周辺にいる僚艦が巻き込まれるかどうかが不明。

### ジャンプ先の位置　**<= 済**

「目標となるジャンプゲート(味方基地)に即時移動」とあるが、基地の位置ぴったりに移動するのか、基地付近に移動するのか。前者だと基地オブジェクトと重なる。

### 旗艦からの補給とジャンプドライブの関係 **<= 済**

艦艇の補給条件は「味方基地または味方旗艦との接触」とあるが、ジャンプドライブはあくまで「味方基地にジャンプ」。旗艦がジャンプゲートになれるかどうかが未定義。

## 5. 数値の妥当性チェック

| 項目 | 計算 | 懸念 |
|------|------|------|
| レーダー範囲(重巡) | 2000 grid = 2.0 座標単位 | 10×10の宇宙でプレーヤーが直径8の範囲を見渡せる。ほぼ全宇宙が見える |
| レーダー範囲(特務艦) | 4000 grid = 4.0 座標単位 | 同上 |
| ミサイル航続距離(一般) | 100 grid/sec × 30 sec = 3000 grid = 3.0 座標単位 | 宇宙の30%をカバー。バランス要検討 |
| ビーム到達距離(特務艦) | 1000 grid = 1.0 座標単位 | レーダー範囲内には十分届く |
| 補給範囲(基地) | 50 grid = 0.05 座標単位 | 非常に狭い。接触に近い感覚 |

## 優先度別まとめ

### 実装前に必須の修正

1. 移動エネルギー消費式の定義
2. ジェネレータ劣化カーブの定義
3. ビームダメージモデルの統一(通過型 vs 爆発範囲型)
4. 衝突半径/物体サイズの定義
5. 勝利条件チェック主体の明示

### 実装中に判断が必要な曖昧さ

1. インテグレータの共有範囲(艦ごと vs 艦隊共有)
2. ビーム発射時の自陣被害の扱い
3. ジャンプ後の着地位置(基地ぴったり vs 近傍)
4. BOTの補給判断閾値

### 軽微な修正(誤字・単位統一)

- `gl` → `gj` (基地耐久性)
- `grit` → `grid` (ミサイル起爆距離)
- 威力の単位 `gj` 追記 (特務艦ミサイル)
- `補給時点` → `補給地点` (BOTコマンダー)
