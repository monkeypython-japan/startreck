## 1. ゲーム概要

### 1.1 目的と勝敗条件

- プレーヤーは特務艦 **U.S.S. YUKIKAZE** の艦長として戦闘に参加する
- ゲームはプレーヤーの入力の有無に関係なく**リアルタイムで進行**する
- **連邦の勝利**: クリンゴン艦隊をすべて撃滅する
- **クリンゴンの勝利**: 連邦の基地をすべて破壊する

### 1.2 座標系

- 宇宙は **0〜10 × 0〜10** の二次元平面（座標値は浮動小数）
- 座標は**閉じた系**: 上限を超えると反対側の下限から出現する（ループ）
- 距離の単位: **1 grid = 0.001 座標単位**
- 宇宙は縦横 1.0 単位ずつの**セクタ**に分割される（合計100セクタ）
- セクタは各軸 0〜9 の組み合わせで呼称する（例: `0-0`, `3-7`, `9-9`）

### 1.3 エネルギーモデル

- エネルギー単位は **gj（ギガジュール）**
- 移動・武装発射・シールド展開・ダメージ・燃料すべてに同単位を使用する
- ジェネレータが燃料を消費してエネルギーを生産し、キャパシタに蓄積する（燃料 1 gj → エネルギー 1 gj）
- **移動コスト**: 速度変化量 (grid/sec) × 1 gj。慣性なし（即座に目標速度へ到達）。キャパシタが不足する場合は速度変更不可（現在速度を維持する）
- 物体は**蓄積ダメージが耐久性を超えた時点**で破壊される

### 1.4 ゲームループ

- 宇宙は毎フレーム（60 FPS）`dt`（秒）ベースで全オブジェクトを更新する
- 累積時間が 1 秒を超えるごとに **1ティック** を発火し、AI（BOTコマンダー）の意思決定を行う
- ティック発火前に破壊フラグをチェックし、破壊されたオブジェクトを宇宙から除去する
- 勝敗判定はゲーム初期化完了後にのみ行う

### 1.5 初期配置ルール

| 対象 | 配置ルール |
|------|-----------|
| 恒星 | 20〜30 個をランダムに配置 |
| クリンゴン艦隊 | 3 艦隊をランダムなセクタ中心に配置。各艦隊は重巡洋艦 1 隻 + 駆逐艦 10 隻（旗艦を中心とした半径 150 grid の円上に等間隔） |
| 連邦基地 | 5 個を互いに離れたセクタ中心に配置 |
| 連邦駆逐艦 | 基地ごとに 10 隻（基地を中心とした半径 150 grid の円上に等間隔） |
| 特務艦 | ランダムに選択された連邦基地の近傍に配置 |
| 全艦艇 | 燃料・キャパシタ・ミサイルが**満載**の状態でスタート |

## 3. オブジェクト仕様

オブジェクトは継承（is-a）または構成（has-a）で関係づける。  
数値はすべて [§2 数値定数一覧](#2-数値定数一覧) を参照する。

```
物体
├── 恒星
├── 基地
└── 移動体
    ├── 艦艇
    │   ├── 重巡洋艦
    │   ├── 駆逐艦
    │   └── 特務艦
    ├── ミサイル
    └── ビーム

装備（艦艇に has-a で保持）
├── ジェネレータ
├── シールド
├── レーダー
├── インテグレータ
├── ブリッジ
├── ミサイルランチャー
│   └── ミサイルナビゲーション（ミサイルに has-a）
├── ビーム発射機
└── ジャンプドライブ（特務艦のみ）
```

---

### 3.1 宇宙

**役割**: 全オブジェクトのコンテナ。ゲームループと勝敗判定を担う。

**保持する状態**

| 状態 | 説明 |
|------|------|
| オブジェクトリスト | 宇宙に存在する全オブジェクト |
| 現在時刻 | ゲーム開始からの秒数（整数） |
| 初期化済みフラグ | `initialize()` 完了後に `true`。勝敗判定はこれが `true` の場合のみ行う |

**振る舞い**

1. 毎フレーム: 破壊フラグを確認し、破壊されたオブジェクトをリストから除去する
2. 毎フレーム: 全オブジェクトの `update(dt)` を呼び出す
3. 毎フレーム: 武器（ミサイル・ビーム）と非武器オブジェクトの衝突を検出してダメージを適用する
4. 1秒ごと: 全艦艇の BOTコマンダーの `tick()` を呼び出す
5. 勝敗判定（初期化済みの場合のみ）:
   - 連邦基地がすべて破壊された → クリンゴンの勝利を宣言
   - クリンゴン艦がすべて破壊された → 連邦の勝利を宣言

---

### 3.2 物体

**役割**: すべてのゲームオブジェクトの基底。

**保持する状態**

| 状態 | 型 | 説明 |
|------|-----|------|
| 位置 | Vec2 | 現在座標 |
| 大きさ | grid | 衝突・ダメージ判定の半径 |
| 耐久性 | gj | 破壊されるまでに吸収できるダメージの上限 |
| 蓄積ダメージ | gj | 累積被ダメージ量 |
| 破壊フラグ | bool | 蓄積ダメージが耐久性を超えると `true` になる |

**振る舞い**

- `receive_damage(amount)`: 蓄積ダメージに加算し、耐久性超過で破壊フラグを立てる。戻り値は実際に船体に与えたダメージ量（シールド吸収後）

---

### 3.3 恒星

**親**: 物体  
**特記**: 耐久性は無限大。いかなるダメージも無効。静止オブジェクト。

---

### 3.4 基地

**親**: 物体  
**特記**: 静止オブジェクト。所属（連邦 `"U"` / クリンゴン `"K"`）を持つ。

**振る舞い**

- 150 grid 以内に接近した味方艦が補給を要求した場合に補給を行う（所要時間 60 sec、補給中は双方移動不可）

---

### 3.5 移動体

**親**: 物体

**保持する状態**

| 状態 | 型 | 説明 |
|------|-----|------|
| 方位ベクトル | Vec2（単位ベクトル） | 現在の進行方向 |
| 現在速度 | grid/sec | |

**振る舞い**

- `update(dt)`: 速度 × dt × GRID 分だけ位置を更新し、座標をラップする
- `accelerate(new_speed)`: 速度を変更し、速度差に応じたエネルギーコストを返す

---

### 3.6 艦艇

**親**: 移動体  
**特記**: 以下の装備を has-a で保持する。

**保持する装備**

| 装備 | 全艦共通 |
|------|---------|
| ジェネレータ | ✓ |
| シールド | ✓ |
| レーダー | ✓ |
| インテグレータ | ✓ |
| ブリッジ | ✓ |
| ミサイルランチャー | ✓ |
| ビーム発射機 | ✓ |
| ジャンプドライブ | 特務艦のみ |

**保持する状態**

| 状態 | 説明 |
|------|------|
| 補給提供フラグ | `true` の艦は僚艦への補給を行う（重巡洋艦のみ `true`） |
| 補給中フラグ | 補給中は移動不可 |
| 所属 | `"U"`: 連邦、`"K"`: クリンゴン |

**振る舞い**

- **ダメージ受信**: シールドが展開中であれば先にシールドで吸収し、残りを艦体ダメージとして蓄積する
- **補給**: 味方基地または味方旗艦に 150 grid 以内で接近して要求した場合、ダメージ全回復・燃料・ミサイルを満載状態に戻す
- **補給提供**: 補給提供フラグが `true` の艦は、コンタクト（150 grid 以内）した僚艦からの要請を受け自艦在庫が 20% を下回らない範囲で補給する

---

### 3.7 重巡洋艦 / 駆逐艦 / 特務艦

各艦種のパラメータは [§2.1 艦種パラメータ](#21-艦種パラメータ) を参照。  
兵装は [§2.2 兵装パラメータ](#22-兵装パラメータ)、シールドは [§2.3](#23-シールドパラメータ) を参照。

| 艦種 | 所属 | 備考 |
|------|------|------|
| 重巡洋艦 | クリンゴン | 艦隊旗艦。補給提供あり |
| 駆逐艦 | 連邦・クリンゴン共通 | BOTコマンダーが搭乗 |
| 特務艦 | 連邦 | プレーヤー専用。ジャンプドライブ搭載 |

---

### 3.8 ミサイル

**親**: 移動体

**保持する状態**

| 状態 | 説明 |
|------|------|
| IFF | `"U"`: 連邦発射、`"K"`: クリンゴン発射 |
| 威力 (gj) | |
| 航続時間 (sec) | 超過すると破壊される |
| ミサイルナビゲーション | has-a |

**振る舞い**

1. `update(dt)`: ミサイルナビゲーションから進路・速度を受け取り移動する
2. 目標の表面から **10 grid** 以内に到達したら起爆し、威力 100% のダメージを与える
3. 起爆または航続時間超過で破壊される

---

### 3.9 ビーム

**親**: 移動体

**保持する状態**

| 状態 | 説明 |
|------|------|
| IFF | `"U"`: 連邦発射、`"K"`: クリンゴン発射 |
| 発射元座標 | ビームの根元（発射機の位置） |
| 威力 (gj) | |
| 到達距離 (grid) | |
| 移動済み距離 (grid) | |

**振る舞い**

1. 発射後、発射元から現在先端位置までの**直線**を形成しながら進む
2. 先端から **10 grid 以内**のオブジェクト: ダメージ 100%
3. 先端から **10〜20 grid** のオブジェクト: ダメージ 25%
4. **発射元艦（オーナー）にはダメージを与えない**（衝突判定から除外）
5. 以下のいずれかで消滅する:
   - 先端がオブジェクトに命中した
   - 先端が恒星の表面に接触した
   - 移動済み距離が到達距離に達した

---

### 3.10 ジェネレータ

**親**: 装備

**保持する状態**

| 状態 | 説明 |
|------|------|
| エネルギー最大発生率 (gj/sec) | |
| 現在発生率 (gj/sec) | |
| キャパシタ最大容量 (gj) | |
| キャパシタ現在量 (gj) | |
| 燃料最大積載量 (gj) | |
| 現在燃料量 (gj) | |

**振る舞い**

- `update(dt)`: 燃料を消費して `発生率 × dt` のエネルギーをキャパシタに充填する（上限: キャパシタ最大容量）
- `request_energy(amount)`: キャパシタから要求量を引き出す。不足時は在庫分のみ提供する
- `consume_energy(amount)`: キャパシタから消費する。不足時は `false` を返す

---

### 3.11 シールド

**親**: 装備

**保持する状態**

| 状態 | 説明 |
|------|------|
| 最大防御エネルギー (gj) | |
| 設定防御率 (%) | プレーヤー/AIが指定する目標値 |
| 現在防御率 (%) | 被弾で低下し、回復率で設定防御率へ戻る |

**振る舞い**

- `absorb(damage)`:
  - 吸収可能量 = 最大防御エネルギー × 現在防御率 / 100
  - 被弾後: 現在防御率 -= 吸収量 / 最大防御エネルギー × 100
  - 戻り値: 艦体に届くダメージ（= 入力ダメージ − 吸収量）
- `update(dt, generator)`:
  - 展開中は `現在防御率 × 展開コスト × dt` のエネルギーを消費する
  - 現在防御率 < 設定防御率 の場合: `回復率 × dt` ぶん回復し、`回復量 × 回復コスト` のエネルギーを消費する

|------|------|

**振る舞い**

### 3.13 インテグレータ

**親**: 装備

**保持する状態**

| 状態 | 説明 |
|------|------|
| 全天マップ | `オブジェクトID → (所属, 最終検出座標, 最終検出時刻)` の辞書 |

**振る舞い**

- `record(obj, game_time)`: レーダーの探索結果を全天マップに追記・更新する
- `query(faction)`: 指定所属のオブジェクト情報を返す
- `remove_destroyed(active_ids)`: 破壊されたオブジェクトの記録を除去する

### 3.15 ミサイルランチャー

**親**: 装備

**保持する状態**

| 状態 | 説明 |
|------|------|
| 最大積載量 | |
| 現在在庫数 | |

**振る舞い**

- `fire(target, radar)`: ミサイルオブジェクトとミサイルナビゲーションを生成して返す

### 3.16 ビーム発射機

**親**: 装備

**振る舞い**

- `fire(target_pos, generator)`:
  - 必要エネルギー = ビーム威力 × 1.25 gj
  - キャパシタ不足時は発射を拒否し、ガンナーへ報告する
  - 成功時はビームオブジェクトを生成して返す

---

### 3.17 ミサイルナビゲーション

**親**: 装備（ミサイルに has-a で保持）

**保持する状態**

| 状態 | 説明 |
|------|------|
| 母艦 | ミサイルを発射した艦艇 |
| 目標オブジェクト | 追尾対象 |

**振る舞い**

- 毎フレーム: 目標が母艦のレーダー探索範囲内にある場合、目標の現在座標に向けて進路ベクトルを更新する
- 目標が母艦のレーダー範囲外の場合: 進路の更新を停止し、最後の進路を維持する
- 母艦が破壊された場合: 進路の更新を停止する

---

### 3.18 ジャンプドライブ

**親**: 装備（特務艦のみ）

**振る舞い**

- `jump(target_base, generator)`:
  - 消費エネルギー = キャパシタ容量の 50%
  - キャパシタ残量が不足している場合は拒否し、プレーヤーへ報告する
  - 成功時: 目標基地を中心とした半径 150 grid の円上のランダムな位置に即時移動する

## 4. 要員仕様

要員はオブジェクト継承ではなく、艦艇のブリッジに搭乗するエージェント。  
NPC艦艇は **BOTコマンダー**、プレーヤー機は **プレーヤー** が搭乗する。

---

### 4.1 ブリッジ要員（基底）

**保持する状態**: 所属艦艇、レーダー、インテグレータへの参照

---

### 4.2 ナビゲーター

**役割**: コマンダーまたは自律判断による移動・回避の実行

**振る舞い**

| メソッド | 説明 |
|---------|------|
| `set_destination(pos, speed)` | 目的地と速度を設定して発進する |
| `stop()` | 即座に停止する |
| `update(dt)` | 目的地に向けて進路を更新し、20 grid 以内で到着とみなして停止する |
| `start_evasion(threat_dir)` | 脅威方向に直交する向きへ最大速度で移動を開始する |
| `end_evasion()` | レーダーから敵ミサイルが消えた時点で回避を終了し停止する |

**報告**: 移動開始・到着・回避機動の開始/継続/終了をメッセージとして出力する

---

### 4.3 ガンナー

**役割**: 武装操作とシールドの自動制御

**振る舞い**

| メソッド / トリガー | 説明 |
|-----------------|------|
| `attack_missile(target)` | 攻撃キューにミサイル攻撃を追加する |
| `attack_beam(target)` | 攻撃キューにビーム攻撃を追加する |
| `update(dt)` | 攻撃キューを順番に処理する。ミサイルは着弾を待たず複数目標へ並行発射可能 |
| 敵ビーム検知 | シールドを自動で **50%** に設定する |
| 敵ミサイル検知（自艦向き） | シールドを自動で **100%** に設定する |

**報告**: 攻撃実行・効果・被弾・シールド状況をメッセージとして出力する。命中報告はシールド吸収量と艦体ダメージを区別して表示する（例: `シールド500gj吸収 / 艦体0gj`）。**自艦が被弾して艦体ダメージが発生した場合**、ガンナーが以下の形式で報告する:
- シールドあり: `被弾: シールドXXgj吸収 / 艦体XXgj ダメージ`
- シールドなし: `被弾: 艦体に XXgj ダメージ`

---

### 4.4 コマンダー（基底）

**役割**: ナビゲーター・ガンナーへの指示を担う基底クラス

- `tick()`: 1秒ごとに呼ばれる意思決定メソッド（サブクラスで実装）

---

### 4.5 BOTコマンダー

**役割**: NPC艦艇の自律 AI。1秒ごとに以下の優先順位で行動を決定する。

**保持する状態**

| 状態 | 説明 |
|------|------|
| ホームオブジェクト | 連邦艦: 所属基地、クリンゴン艦: 所属艦隊の旗艦 |

**tick() の優先順位**

1. **退避判定**: 蓄積ダメージ ≥ 耐久性の 80% → ホームオブジェクトへ最大速度で移動する
2. **敵探索**: インテグレータからホームに最も近い敵を取得する
3. **攻撃（ビーム優先）**:
   - 敵が **ビーム射程内**: ガンナーにビーム攻撃を指示する
   - 敵が **ミサイル射程内**: ガンナーにミサイル攻撃を指示する
4. **接近**: 敵が射程外 → ナビゲーターに最大速度で接近を指示する
5. **索敵移動**: 敵が見つからない → ホームオブジェクトから離れる方向へ最大速度で移動し続ける。毎秒方向を再計算するため、ホームから遠ざかりながら索敵を続ける。敵を発見するまで移動は止まらない

**注意**: 攻撃を検知した場合はナビゲーターに回避機動を指示する（優先度は上記に割り込まない）

---

### 4.6 プレーヤー

**役割**: 人間プレーヤーのアバター。UIイベントをクルーへの指示に変換する。

**保持する状態**: UIへの参照

**振る舞い**

| メソッド | 説明 |
|---------|------|
| `move_to(pos, speed)` | ナビゲーターに移動を指示する |
| `stop()` | ナビゲーターに停止を指示する |
| `attack_missile(target)` | ガンナーにミサイル攻撃を指示する |
| `attack_beam(target)` | ガンナーにビーム攻撃を指示する |
| `set_shield(rate)` | シールドの設定防御率を変更する |
| `jump_to(base)` | ジャンプドライブを起動する |

## 5. UI仕様

> **変更指針**: UIの追加・変更はこのセクションのみを修正する。  
> 実装クラスは `game/ui/` 以下に配置する。

### 5.1 レイアウト

```
┌──────────────────────────────┬──────────────────────┐
│                              │  レーダービュー       │
│       全天マップ              │  470 × 470 px        │
│       960 × 960 px           │  白背景 (§5.3)       │
│       暗背景 (§5.2)          ├──────────────────────┤
│       プレーヤー中央固定     │  ステータスパネル     │
│                              │  (§5.4)              │
│                              ├──────────────────────┤
│                              │  メッセージウィンドウ │
│                              │  (§5.5)              │
└──────────────────────────────┴──────────────────────┘
   ウィンドウ全体: 1440 × 960 px
```

### 5.2 全天マップ

インテグレータに記録された全オブジェクトを表示する広域ビュー。

- 宇宙の 0〜10 × 0〜10 座標を **960 × 960 px** の正方形に 1:1 で写像する
- **プレーヤー艦を常に中央に固定**する。艦の移動に追従してスクロールし、ジャンプ時は瞬時に切り替える
- セクタ境界を半透明の**破線グリッド**で表示する
- プレーヤー中心に以下の範囲円を描画する:
  - **レーダー探知範囲**: 破線円
  - **ミサイル射程**: 半透明の赤い塗りつぶし円
  - **ビーム射程**: 半透明の青い塗りつぶし円
- マウスクリックによるコマンド入力に対応する（§5.6 参照）

**オブジェクトの表示**

| オブジェクト | 色（暗背景） | 形 |
|------------|------------|-----|
| 自艦 | 黄 | 丸 |
| 僚艦（連邦） | 青 | 丸 |
| 敵艦（クリンゴン） | 赤 | 丸 |
| 連邦基地 | オレンジ | 丸 |
| 恒星 | 黄緑 | 丸 |
| 味方ミサイル | 緑 | 丸 |
| 敵ミサイル | 橙 | 丸 |
| 味方ビーム | 水色 | 線 |
| 敵ビーム | 赤 | 線 |

- 移動体には速度・進行方向に対応した矢印を付与する
- 選択中のオブジェクトはハイライト表示する
- 自艦のミサイル・ビームはレーダーレンジ外でも表示する
- ビームの軌跡は徐々に薄くなる余韻表示を行う（未実装）

### 5.3 レーダービュー

現在のレーダースキャン結果をズーム表示するサブビュー。

- **白背景**、サイズ **470 × 470 px**（正方形）
- プレーヤー艦を中央に固定する
- **レーダー探知範囲がビュー全体に対応**するようスケーリングする（探知範囲端 = ビュー端）
- 探知範囲の境界を**破線の円**として描画する
- 表示するオブジェクトは現在のレーダースキャン結果のみ
- セクタ境界を半透明の**破線グリッド**で表示する
- オブジェクトの色は白背景に映えるよう全天マップより暗めの配色とする
- マウスクリックによるコマンド入力に対応する（§5.6 参照）

### 5.4 ステータスパネル

リアルタイムで以下の情報をバー表示する。

| 項目 | 表示形式 |
|------|---------|
| 現在座標・セクタ | テキスト |
| 速度 | テキスト（現在値 / 最大値） |
| HP（耐久性 − 蓄積ダメージ） | バー + 数値 |
| シールド現在防御率 / 設定防御率 | バー + 数値 |
| キャパシタ現在量 / 最大容量 | バー + 数値 |
| 燃料残量 / 最大積載量 | バー + 数値 |
| ミサイル残弾 / 積載量 | テキスト |
| ジャンプドライブ状態 | READY / 不足エネルギー量 |
| ゲーム経過時間 / オブジェクト総数 | テキスト |
| 連邦基地残数 / クリンゴン残数 | テキスト |

### 5.5 メッセージウィンドウ

- ナビゲーター・ガンナーからの報告を日本語テキストで時系列に表示する
- 最新メッセージが常に末尾に追加され、ウィンドウは自動スクロールする
- マウスホイールで過去のログを遡ることができる
- 最大保持件数: 300 件

### 5.6 ポップアップメニュー

全天マップまたはレーダービュー上でオブジェクトをクリックすると、対象に応じたコンテキストメニューが表示される。両ビューで同一の操作が可能。

**空白エリアをクリック**

| メニュー項目 | 動作 |
|------------|------|
| 最大速度で移動 | ナビゲーターに最大速度で移動指示 |
| 半速で移動 | ナビゲーターに最大速度の 50% で移動指示 |
| 低速で移動 | ナビゲーターに最大速度の 25% で移動指示 |
| 停止 | ナビゲーターに停止指示 |

**敵艦をクリック**

| メニュー項目 | 有効条件 | 動作 |
|------------|---------|------|
| ミサイル攻撃 | ミサイル残弾あり | ガンナーにミサイル攻撃を指示 |
| ビーム攻撃 | 常時 | ガンナーにビーム攻撃を指示 |
| この目標へ移動 | 常時 | 移動速度サブメニューを表示 |

**連邦基地をクリック**

| メニュー項目 | 有効条件 | 動作 |
|------------|---------|------|
| この基地へ移動 | 常時 | 移動速度サブメニューを表示 |
| ジャンプ | エネルギー充足 | ジャンプドライブを起動 |

**自艦をクリック**

| メニュー項目 | 動作 |
|------------|------|
| 停止 | ナビゲーターに停止指示 |
| シールド 0% | シールドを解除 |
| シールド 50% | シールドを 50% に設定 |
| シールド 100% | シールドを 100% に設定 |
| ジャンプ先を選択 | 次のクリックでジャンプ先基地を選択するモードへ移行 |

### 5.7 キーボードショートカット

| キー | 動作 |
|------|------|
| `Space` | 即時停止 |
| `S` | シールドのオン（100%）/ オフ（0%）トグル |
| `ESC` | ゲーム終了 |
