## 1. ゲーム概要

### 1.1 目的と勝敗条件

- プレーヤーは特務艦 **U.S.S. YUKIKAZE** の艦長として戦闘に参加する
- ゲームはプレーヤーの入力の有無に関係なく**リアルタイムで進行**する
- **連邦の勝利**: クリンゴンの基地をすべて破壊する
- **クリンゴンの勝利**: 連邦の基地をすべて破壊する

### 1.2 座標系

- 宇宙は **0〜10 × 0〜10** の二次元平面（座標値は浮動小数）
- 座標は**閉じた系**: 上限を超えると反対側の下限から出現する（ループ）
- 距離の単位: **1 grid = 0.001 座標単位**
- 宇宙は縦横 1.0 単位ずつの**セクタ**に分割される（合計100セクタ）
- セクタは各軸 0〜9 の組み合わせで呼称する（例: `0-0`, `3-7`, `9-9`）

### 1.3 エネルギーモデル

- エネルギー単位は **gj（ギガジュール）**
- 移動・武装発射・シールド展開・ダメージ・燃料すべてに同単位を使用する
- ジェネレータが燃料を消費してエネルギーを生産し、キャパシタに蓄積する（燃料 1 gj → エネルギー 1 gj）
- **移動コスト**: 速度変化量 (grid/sec) × 1 gj。慣性なし（即座に目標速度へ到達）。キャパシタが不足する場合は速度変更不可（現在速度を維持する）
- 物体は**蓄積ダメージが耐久性を超えた時点**で破壊される

### 1.4 ゲームループ

- 宇宙は毎フレーム（60 FPS）`dt`（秒）ベースで全オブジェクトを更新する
- 累積時間が 1 秒を超えるごとに **1ティック** を発火し、AI（BOTコマンダー）の意思決定を行う
- ティック発火前に破壊フラグをチェックし、破壊されたオブジェクトを宇宙から除去する
- 勝敗判定はゲーム初期化完了後にのみ行う

### 1.5 初期配置ルール

| 対象 | 配置ルール |
|------|-----------|
| 恒星 | 10〜15 個をジッタードグリッド法（4×4 グリッドのセルからランダムに選択し、セル内でランダム配置）で宇宙全体に均等分布させる |
| 連邦基地 | 5 個を **x セクタ 0〜4** 内のセクタ中心に配置。同勢力の基地同士は **3000 grid 以上**離す |
| 連邦艦隊 | 3 艦隊を担当基地（ラウンドロビンで割り当て）から半径 300 grid の円上にランダム配置。各艦隊は重巡洋艦 1 隻（フリートコマンダー）+ 駆逐艦 10 隻（旗艦を中心とした半径 150 grid の円上に等間隔）。旗艦のホームは担当基地 |
| クリンゴン基地 | 5 個を **x セクタ 5〜9** 内のセクタ中心に配置。同勢力の基地同士は **3000 grid 以上**離す |
| クリンゴン艦隊 | 3 艦隊を担当基地（ラウンドロビンで割り当て）から半径 300 grid の円上にランダム配置。各艦隊は重巡洋艦 1 隻（フリートコマンダー）+ 駆逐艦 10 隻（旗艦を中心とした半径 150 grid の円上に等間隔）。旗艦のホームは担当基地 |
| 連邦護衛型駆逐艦 | 連邦基地 1 個につき 10 隻を基地から半径 600 grid の円上に等間隔配置（計 50 隻）。連邦旗艦 1 隻につき 5 隻を旗艦から半径 600 grid の円上に等間隔配置（計 15 隻）。合計 65 隻。各艦のホームは配置元（基地または旗艦） |
| クリンゴン護衛型駆逐艦 | 連邦護衛型駆逐艦と同様（クリンゴン基地・旗艦を対象）。合計 65 隻 |
| 特務艦 | ランダムに選択された連邦基地の近傍に配置 |
| 全艦艇 | 燃料・キャパシタ・ミサイルが**満載**の状態でスタート |

## 3. オブジェクト仕様

オブジェクトは継承（is-a）または構成（has-a）で関係づける。  
数値はすべて [§2 数値定数一覧](#2-数値定数一覧) を参照する。

```
物体
├── 恒星
├── 基地
└── 移動体
    ├── 艦艇
    │   ├── 重巡洋艦
    │   │   └── 護衛型駆逐艦  (駆逐艦のサブクラス、BOTガードコマンダーが搭乗)
    │   ├── 駆逐艦
    │   └── 特務艦
    ├── ミサイル
    └── ビーム

装備（艦艇に has-a で保持）
├── ジェネレータ
├── シールド
├── レーダー
├── インテグレータ
├── ブリッジ
├── ミサイルランチャー
│   └── ミサイルナビゲーション（ミサイルに has-a）
├── ビーム発射機
└── ジャンプドライブ（特務艦のみ）
```

---

### 3.1 宇宙

**役割**: 全オブジェクトのコンテナ。ゲームループと勝敗判定を担う。

**保持する状態**

| 状態 | 説明 |
|------|------|
| オブジェクトリスト | 宇宙に存在する全オブジェクト |
| 現在時刻 | ゲーム開始からの秒数（整数） |
| 初期化済みフラグ | `initialize()` 完了後に `true`。勝敗判定はこれが `true` の場合のみ行う |

**振る舞い**

1. 毎フレーム: 破壊フラグを確認し、破壊されたオブジェクトをリストから除去する
2. 毎フレーム: 全オブジェクトの `update(dt)` を呼び出す
3. 毎フレーム: 武器（ミサイル・ビーム）と非武器オブジェクトの衝突を検出してダメージを適用する
4. 1秒ごと: 全艦艇の BOTコマンダーの `tick()` を呼び出す
5. 勝敗判定（初期化済みの場合のみ）:
   - 連邦基地がすべて破壊された → クリンゴンの勝利を宣言
   - クリンゴン基地がすべて破壊された → 連邦の勝利を宣言
   - 勝利確定後はウィンドウを閉じず、勝利メッセージを画面中央に表示したままゲームを停止する。プレーヤーが明示的に終了操作を行うまで待機する

---

### 3.2 物体

**役割**: すべてのゲームオブジェクトの基底。

**保持する状態**

| 状態 | 型 | 説明 |
|------|-----|------|
| 位置 | Vec2 | 現在座標 |
| 大きさ | grid | 衝突・ダメージ判定の半径 |
| 耐久性 | gj | 破壊されるまでに吸収できるダメージの上限 |
| 蓄積ダメージ | gj | 累積被ダメージ量 |
| 破壊フラグ | bool | 蓄積ダメージが耐久性を超えると `true` になる |

**振る舞い**

- `receive_damage(amount)`: 蓄積ダメージに加算し、耐久性超過で破壊フラグを立てる。戻り値は実際に船体に与えたダメージ量（シールド吸収後）

---

### 3.3 恒星

**親**: 物体  
**特記**: 耐久性は無限大。いかなるダメージも無効。静止オブジェクト。

---

### 3.4 基地

**親**: 物体  
**特記**: 静止オブジェクト。所属（連邦 `"U"` / クリンゴン `"K"`）を持つ。レーダー（レンジ 1000 grid）とインテグレータを保持する。

**振る舞い**

- 150 grid 以内に接近した味方艦が補給を要求した場合に補給を行う（所要時間 60 sec、補給中は双方移動不可）
- レーダーは毎ティックスキャンし、探知した敵情報を同勢力の全艦艇インテグレータに共有する

---

### 3.5 移動体

**親**: 物体

**保持する状態**

| 状態 | 型 | 説明 |
|------|-----|------|
| 方位ベクトル | Vec2（単位ベクトル） | 現在の進行方向 |
| 現在速度 | grid/sec | |

**振る舞い**

- `update(dt)`: 速度 × dt × GRID 分だけ位置を更新し、座標をラップする
- `accelerate(new_speed)`: 速度を変更し、速度差に応じたエネルギーコストを返す

---

### 3.6 艦艇

**親**: 移動体  
**特記**: 以下の装備を has-a で保持する。

**保持する装備**

| 装備 | 全艦共通 |
|------|---------|
| ジェネレータ | ✓ |
| シールド | ✓ |
| レーダー | ✓ |
| インテグレータ | ✓ |
| ブリッジ | ✓ |
| ミサイルランチャー | ✓ |
| ビーム発射機 | ✓ |
| ジャンプドライブ | 特務艦のみ |

**保持する状態**

| 状態 | 説明 |
|------|------|
| 補給提供フラグ | `true` の艦は僚艦への補給を行う（重巡洋艦のみ `true`） |
| 補給中フラグ | 補給中は移動不可 |
| 所属 | `"U"`: 連邦、`"K"`: クリンゴン |

**振る舞い**

- **ダメージ受信**: シールドが展開中であれば先にシールドで吸収し、残りを艦体ダメージとして蓄積する
- **補給**: 味方基地または味方旗艦に 150 grid 以内で接近して要求した場合、ダメージ全回復・燃料・ミサイルを満載状態に戻す
- **補給提供**: 補給提供フラグが `true` の艦は、コンタクト（150 grid 以内）した僚艦からの要請を受け自艦在庫が 20% を下回らない範囲で補給する

| 護衛型駆逐艦 | 連邦・クリンゴン共通 | 駆逐艦のサブクラス。BOTガードコマンダーが搭乗。ホームの基地または旗艦を防衛する |
| 駆逐艦 | 連邦・クリンゴン共通 | BOTコマンダーが搭乗 |

### 3.8 ミサイル

**親**: 移動体

**保持する状態**

| 状態 | 説明 |
|------|------|
| IFF | `"U"`: 連邦発射、`"K"`: クリンゴン発射 |
| 威力 (gj) | |
| 航続時間 (sec) | 超過すると破壊される |
| ミサイルナビゲーション | has-a |

**振る舞い**

1. `update(dt)`: ミサイルナビゲーションから進路・速度を受け取り移動する
2. 目標の表面から **10 grid** 以内に到達したら起爆し、威力 100% のダメージを与える
3. 起爆または航続時間超過で破壊される

---

### 3.9 ビーム

**親**: 移動体

**保持する状態**

| 状態 | 説明 |
|------|------|
| IFF | `"U"`: 連邦発射、`"K"`: クリンゴン発射 |
| 発射元座標 | ビームの根元（発射機の位置） |
| 威力 (gj) | |
| 到達距離 (grid) | |
| 移動済み距離 (grid) | |

**振る舞い**

1. 発射後、発射元から現在先端位置までの**直線**を形成しながら進む
2. 先端から **10 grid 以内**のオブジェクト: ダメージ 100%
3. 先端から **10〜20 grid** のオブジェクト: ダメージ 25%
4. **発射元艦（オーナー）にはダメージを与えない**（衝突判定から除外）
5. 以下のいずれかで消滅する:
   - 先端がオブジェクトに命中した
   - 先端が恒星の表面に接触した
   - 移動済み距離が到達距離に達した

---

### 3.10 ジェネレータ

**親**: 装備

**保持する状態**

| 状態 | 説明 |
|------|------|
| エネルギー最大発生率 (gj/sec) | |
| 現在発生率 (gj/sec) | |
| キャパシタ最大容量 (gj) | |
| キャパシタ現在量 (gj) | |
| 燃料最大積載量 (gj) | |
| 現在燃料量 (gj) | |

**振る舞い**

- `update(dt)`: 燃料を消費して `発生率 × dt` のエネルギーをキャパシタに充填する（上限: キャパシタ最大容量）
- `request_energy(amount)`: キャパシタから要求量を引き出す。不足時は在庫分のみ提供する
- `consume_energy(amount)`: キャパシタから消費する。不足時は `false` を返す

---

### 3.11 シールド

**親**: 装備

**保持する状態**

| 状態 | 説明 |
|------|------|
| 最大防御エネルギー (gj) | |
| 設定防御率 (%) | プレーヤー/AIが指定する目標値 |
| 現在防御率 (%) | 被弾で低下し、回復率で設定防御率へ戻る |

**振る舞い**

- `absorb(damage)`:
  - 吸収可能量 = 最大防御エネルギー × 現在防御率 / 100
  - 被弾後: 現在防御率 -= 吸収量 / 最大防御エネルギー × 100
  - 戻り値: 艦体に届くダメージ（= 入力ダメージ − 吸収量）
- `update(dt, generator)`:
  - 展開中は `現在防御率 × 展開コスト × dt` のエネルギーを消費する
  - 現在防御率 < 設定防御率 の場合: `回復率 × dt` ぶん回復し、`回復量 × 回復コスト` のエネルギーを消費する

---

### 3.12 レーダー

**親**: 装備

**保持する状態**

| 状態 | 説明 |
|------|------|
| 探索範囲 (grid) | |
| コンタクトリスト | 探索範囲内のオブジェクト |

**振る舞い**

- `scan(all_objects, game_time)`: 探索範囲内のオブジェクトを検出し、インテグレータに報告する

### 3.13 インテグレータ

**親**: 装備  
**特記**: 同勢力の全艦艇でデータを共有する全天マップ。敵勢力には共有しない。

**保持する状態**

| 状態 | 説明 |
|------|------|
| 全天マップ | `オブジェクトID → (所属, 最終検出座標, 最終検出時刻)` の辞書 |

**データ更新規則**

インテグレータは敵味方ともに同じ仕様で動作する。勢力を問わず全艦艇が以下の規則に従ってインテグレータを保持・更新する。

| オブジェクト種別 | 更新タイミング |
|-----------------|---------------|
| 恒星 | ゲーム開始時から双方のインテグレータに記録済み。以降も毎秒常時反映 |
| 同勢力の艦艇・基地 | 毎秒 (1ティック)、現在座標・進路で常時反映 |
| 敵艦 | いずれかの味方艦レーダーに捕捉されているときのみ更新 |

**振る舞い**

- `record(obj, game_time)`: レーダーの探索結果を全天マップに追記・更新する
- `query(faction)`: 指定所属のオブジェクト情報を返す
- `remove_destroyed(active_ids)`: 破壊されたオブジェクトの記録を除去する

**フリート同期** (`Universe._tick()` 内で毎秒実行、ゲーム開始時にも一度実行)

1. 破壊済みオブジェクトの記録を除去する
2. 全恒星を全艦艇のインテグレータに記録する（双方に既知）
3. 同勢力の全オブジェクトを現在座標で記録する（常時反映）
4. いずれかの味方レーダーが捕捉している敵オブジェクトを全味方艦のインテグレータに記録する（レーダー共有）

### 4.1 ブリッジ要員（基底）

**保持する状態**: 所属艦艇、レーダー、インテグレータへの参照

### 4.2 ナビゲーター

**役割**: コマンダーまたは自律判断による移動・回避の実行

**振る舞い**

| メソッド | 説明 |
|---------|------|
| `set_destination(pos, speed)` | 目的地と速度を設定して発進する |
| `stop()` | 即座に停止する |
| `update(dt)` | 目的地に向けて進路を更新し、20 grid 以内で到着とみなして停止する。移動中は分離ステアリングを適用する |
| `start_evasion(threat_dir)` | 脅威方向に直交する向きへ最大速度で移動を開始する（ミサイル回避用） |
| `end_evasion()` | レーダーから敵ミサイルが消えた時点で回避を終了し停止する |
| `start_flee_evasion()` | コマンダー指示による敵密度回避を開始する |
| `end_flee_evasion()` | レーダーから敵艦が消えた時点で回避を終了し、コマンダーへ通知する |

**分離ステアリング**:
- 移動中（速度 > 0、回避機動中を除く）、レーダーコンタクト内の艦艇との距離を確認する
- 10 grid 未満: 距離に反比例した強い反発ベクトルで進路を調整する
- 10〜30 grid かつ進路前方 53° 以内: 弱い回避ベクトルを適用する
- 反発ベクトルと現在 heading が完全に逆向きになる場合は直角方向へ偏向する

**敵密度回避の動作**:
- 毎フレーム、レーダー範囲内の敵艦の重心方向を計算し、その逆方向を進路として最大速度で移動する
- 全方位から囲まれている場合は最遠の敵の反対側へ向かう
- レーダーから敵艦が消えた時点で自動終了し、コマンダーへ報告する

**報告**: 移動開始・到着・回避行動の開始/終了をメッセージとして出力する

---

### 4.3 ガンナー

**役割**: 武装操作とシールドの自動制御

**振る舞い**

| メソッド / トリガー | 説明 |
|-----------------|------|
| `attack_missile(target)` | 攻撃キューにミサイル攻撃を追加する |
| `attack_beam(target)` | 攻撃キューにビーム攻撃を追加する |
| `update(dt)` | 攻撃キューを順番に処理する。ミサイルは着弾を待たず複数目標へ並行発射可能。**射程外の目標には発射しない**（ミサイル: speed×flight_time、ビーム: beam_range を超える場合はキャンセルして射程外を報告） |
| 敵ビーム検知 | シールドを自動で **50%** に設定する |
| 敵ミサイル検知（自艦向き） | シールドを自動で **100%** に設定する |
| 脅威なし・コマンダー指示なし | シールドを **0%** に設定する（`_manual_shield_rate == 0` のとき） |

**報告**: 攻撃実行・効果・被弾・シールド状況をメッセージとして出力する。命中報告はシールド吸収量と艦体ダメージを区別して表示する（例: `シールド500gj吸収 / 艦体0gj`）。**自艦が被弾して艦体ダメージが発生した場合**、ガンナーが以下の形式で報告する:
- シールドあり: `被弾: シールドXXgj吸収 / 艦体XXgj ダメージ`
- シールドなし: `被弾: 艦体に XXgj ダメージ`

**撃沈報告**: 攻撃で目標が破壊された場合、武器（ミサイル・ビーム）が `[目標クラス名] 撃沈` を**赤字**でクルーログに表示する。

### 4.5 BOTコマンダー

**役割**: NPC艦艇（駆逐艦）の自律 AI。1秒ごとに以下の優先順位で行動を決定する。

**保持する状態**

| 状態 | 説明 |
|------|------|
| ホームオブジェクト | 所属艦隊の旗艦（両軍共通）。旗艦破壊時は最近傍の味方補給提供艦に再割り当て |
| 攻撃者ID | 直近で攻撃してきた敵艦/基地のID（被弾時にセット） |
| 割り当て基地ID | フリートコマンダーから指示された攻撃目標基地のID |

**tick() の優先順位**

0. **ホーム再割り当て**: ホームオブジェクト（旗艦）が破壊されている場合、生き残った同勢力の旗艦のうち最寄りの `BotFleetCommander` に編入リクエストを送る。フリートコマンダーは即座に承認し、その旗艦を新しいホームに設定する（候補がなければホームを `None` にする）
0. **シールド管理**: 自艦に向かう敵武器がレーダー範囲内にない場合、ガンナーにシールド 0% を指示する（エネルギー節約）
1. **退避判定**: 蓄積ダメージ ≥ 耐久性の 80% → 最寄りの味方補給地点（基地または補給フラグ艦）へ最大速度で向かう。補給範囲（150 grid）以内に入ったら即座に `supply_full()` を実行し、補給完了後は直ちに索敵移動（優先度 6 と同じ挙動）を開始する
2. **回避行動**: 蓄積ダメージ ≥ 耐久性の 70% かつ 80% 未満 → ナビゲーターに回避行動を指示する。回避中はこの優先度から戻らない（ナビゲーターが終了を通知するまで）
3. **攻撃目標選択**（以下の優先順位）:
   - **攻撃者への反撃**: 直近で攻撃してきた敵がインテグレータに存在すれば優先して攻撃する
   - **割り当て基地**: フリートコマンダーから指示された敵基地がインテグレータに存在すれば攻撃する
   - **最近傍の敵基地**: インテグレータ内の最近傍の敵 `BaseStation` を攻撃する
   - **最近傍の敵艦**: インテグレータ内の最近傍の敵艦を攻撃する
4. **攻撃（ミサイル優先）**:
   - 敵が **ミサイル射程内**: ガンナーにミサイル攻撃を指示する（ミサイル優先）
   - 敵が **ビーム射程内かつミサイル射程外**: ガンナーにビーム攻撃を指示する
5. **接近**: 敵が射程外 → ナビゲーターに最大速度で接近を指示する
6. **索敵移動**: 敵が見つからない → ホームオブジェクトから離れる方向へ最大速度で移動し続ける。毎秒方向を再計算するため、ホームから遠ざかりながら索敵を続ける。敵を発見するまで移動は止まらない

**注意**: 攻撃を検知した場合はナビゲーターに回避機動を指示する（優先度は上記に割り込まない）

---

### 4.5.1 BOTフリートコマンダー

**役割**: 各艦隊の旗艦（重巡洋艦）に搭乗する AI。BOTコマンダーを継承し、艦隊指揮機能を追加する。

**保持する状態**

| 状態 | 説明 |
|------|------|
| 艦隊メンバー | 傘下の駆逐艦リスト |

**tick() の追加動作** (super().tick() の前に実行)

1. `update_fleet_target()` を呼び出して艦隊目標を更新する
2. その後 `super().tick()` で通常の BOTコマンダー行動を実行する

**`update_fleet_target()`**

- インテグレータから最近傍の敵基地を選択し、自身の割り当て基地IDにセットする
- 艦隊メンバー全員の `set_attack_target()` を呼び出して同じ基地IDを通知する
- インテグレータに敵基地が新規登録または消滅した際も `universe` から即時呼び出される（tick 待ちなし）

**編入リクエスト処理**

- 同勢力の駆逐艦から `accept_join_request(vessel)` を受け取った場合、即座に承認して艦隊メンバーに追加する（拒否しない）

---

### 4.5.2 BOTガードコマンダー

**役割**: 護衛型駆逐艦のホーム（基地または旗艦）防衛に特化した自律 AI。BOTコマンダーを継承する。

**保持する状態**

| 状態 | 説明 |
|------|------|
| ホーム | 防衛対象の基地または旗艦（BotFleetCommander搭乗艦） |

**tick() の動作** (優先順位順)

1. ホームが破壊されていれば最近傍の同勢力基地または旗艦に再割り当て
2. **補給判定** (`_needs_supply()`)が真なら補給先へ移動（以降の処理をスキップ）
3. 蓄積ダメージが耐久性の **70%** 以上なら逃避回避行動（以降の処理をスキップ）
4. 攻撃目標を選択（`_select_guard_target()`）
   - 目標なし → `_guard_position()` でホーム周辺の待機位置へ移動
   - 目標あり → 目標がホームから 1300 grid 超かつ射程外なら追跡しない（`_guard_position()` へ）
   - ミサイル射程内 → ミサイル攻撃
   - ビーム射程内 → ビーム攻撃
   - 射程外 → 目標へ接近

**補給判定 (`_needs_supply()`)**

ホームが存在し破壊されていない場合に以下いずれかが満たされると `True`:
- 蓄積ダメージ ≥ 耐久性 × 35%
- 燃料 < 燃料最大 × 30%
- ミサイル残弾 < 積載量 × 30%

ホームが存在しないまたは破壊済みの場合は常に `False`（補給行動なし）

**補給行動 (`_go_supply()`)**

ホーム（基地）が補給範囲内なら `supply_full()` で即時補給。範囲外ならホームへ移動。

**攻撃目標選択 (`_select_guard_target()`)** (優先順位順)

1. **ホームへの攻撃者**: レーダーコンタクト中の敵兵器（ミサイル/ビーム）がホームへ向かっている（内積 > 0.5）場合、その発射元艦艇レコードを返す
2. **自艦への攻撃者**: `on_attacked_by()` で記録された攻撃者
3. **ホームへ接近中の敵**: ホームから 600 grid 以内でホーム方向へ移動中（内積 > 0.3）の敵艦艇。最もホームから遠い脅威を優先（ホームから離れた位置で排除するため）
4. **最近傍の敵基地**: インテグレータに記録された最も近い敵基地

**待機位置管理 (`_guard_position()`)**

- ホームが**基地**（静止）の場合: 距離が 200〜1000 grid の範囲外のときのみ、ホームから **600 grid** の位置（ホーム→自艦方向）へ移動する
- ホームが**旗艦**（移動体）の場合: 距離に関わらず毎ティック目的地を更新し、旗艦から **600 grid** の位置を維持しながら追従する

**ホーム再割り当て (`_reassign_home()`)**

ホームが破壊された時、同勢力の基地または BotFleetCommander 搭乗艦の中から最近傍を新たなホームに設定する。候補がなければ `None`（以降は補給なし・待機行動のみ）。

### 5.2 全天マップ

フリートインテグレータに記録されたオブジェクトを表示する広域ビュー。同勢力オブジェクトは常に表示され、敵・中立オブジェクトはいずれかの味方レーダーに捕捉された場合のみ表示される。

- 宇宙の 0〜10 × 0〜10 座標を **960 × 960 px** の正方形に 1:1 で写像する
- **プレーヤー艦を常に中央に固定**する。艦の移動に追従してスクロールする
- **ジャンプ演出**: ジャンプ実行時は、ジャンプ前の位置からジャンプ後の位置へ向かって全天マップが 0.5 秒かけて高速スクロールし、ジャンプ方向をプレーヤーに示す
- **アクティブレーダー捕捉表示**: 自艦または僚艦のレーダーに現在捕捉されている**敵オブジェクト**の中心に白い小点（半径 2px）を表示する。僚艦・味方基地・恒星には表示しない。いずれのレーダーでも捕捉されなくなると白点は消える
- **敵探知済み味方基地**: 敵艦艇のインテグレータに記録されている味方基地の中心に白い小点（半径 2px）を表示する
- **敵レーダー捕捉中の味方艦艇**: 敵艦艇のレーダーに現在捕捉されている味方艦艇の中心に白い小点（半径 2px）を表示する。レーダー範囲外に出ると白点は消える
- **スキャン済みセクターオーバーレイ**: プレーヤーのレーダーが全域をカバーしたセクターに半透明グレーを重ねて表示する。セクターの4隅すべてがレーダー範囲内に入った時点でスキャン済みとなり、以降は永続的に表示される
- セクタ境界を半透明の**破線グリッド**で表示する
- プレーヤー中心に以下の範囲円を描画する:
  - **レーダー探知範囲**: 破線円
  - **ミサイル射程**: 半透明の赤い塗りつぶし円
  - **ビーム射程**: 半透明の青い塗りつぶし円
- マウスクリックによるコマンド入力に対応する（§5.6 参照）
- **移動中マーカー**: ナビゲーターが目的地へ移動中、目的地に水色の十字マーカーを表示し、自艦位置からマーカーまで薄い破線を引く。到着後はマーカーと破線を消去する

**オブジェクトの表示**

| オブジェクト | 色（暗背景） | 形 |
|------------|------------|-----|
| 自艦 | 黄 | 丸（大） |
| 僚艦（連邦） | 青 | 丸 |
| 連邦旗艦 | 青 | 二重丸（大） |

### 5.3 レーダービュー

現在のレーダースキャン結果をズーム表示するサブビュー。

- **白背景**、サイズ **470 × 470 px**（正方形）
- プレーヤー艦を中央に固定する
- **レーダー探知範囲がビュー全体に対応**するようスケーリングする（探知範囲端 = ビュー端）
- マウスカーソルがレーダービュー内にある場合、マウスホイールで表示レンジを変更してズームできる（1〜3倍、ステップ 0.25）。3倍時は探知範囲の 1/3 がビューに表示される
- 探知範囲の境界を**破線の円**として描画する
- **ミサイル射程**: ミサイル残弾がある場合、プレーヤー中心に半透明の青い塗りつぶし円を表示する
- 表示するオブジェクトは現在のレーダースキャン結果のみ
- **移動中マーカー**: 全天マップと同様に、目的地の十字マーカーと薄い破線を表示する。目的地がレーダービュー範囲外の場合は破線をビュー端まで描画する（十字マーカーは表示しない）
- セクタ境界を半透明の**破線グリッド**で表示し、各セクターの左上隅に `X,Y` 形式のセクター番号を小さく表記する
- オブジェクトの色は白背景に映えるよう全天マップより暗めの配色とする
- **自艦撃沈時**: 赤いバツ印を表示する（全天マップと同様）
- マウスクリックによるコマンド入力に対応する（§5.6 参照）

### 5.6 ポップアップメニュー

全天マップ上でオブジェクトをクリックするとコンテキストメニューが表示される。レーダービューは敵オブジェクトのクリック時のみ動作が異なる（下記参照）。

**レーダービューで敵艦・敵基地をクリック（ポップアップなし・即時発射）**

ポップアップを表示せず、以下のルールで攻撃指示を即時発行する:

| 条件 | 動作 |
|------|------|
| ミサイル残弾あり かつ 目標がミサイル射程内 | ガンナーにミサイル攻撃を指示 |
| 上記以外 | ガンナーにビーム攻撃を指示 |

### 5.7 コントロールバー

右パネル最下部に常時表示される 3 つのボタン。

| ボタン | 状態による表示変化 | 動作 |
|--------|-------------------|------|
| スタート/一時停止/再開 | 待機中: `▶ スタート` / 実行中: `一時停止` / 停止中: `▶ 再開` | ゲームの実行状態をトグルする。初回クリックでゲームを開始し、以降は一時停止・再開を交互に切り替える |
| リセット | 常に `リセット` | ゲームを最初から再開する（新しい宇宙を生成） |
| 終了 | 常に `終了` | ゲームを終了しアプリを閉じる |

**ゲーム開始状態**: 起動直後は **待機中**（宇宙は更新されない）。`▶ スタート` をクリックしてゲームを開始する。

### 5.8 キーボードショートカット

| キー | 動作 |
|------|------|
| `Space` | 即時停止 |
| `S` | シールドのオン（100%）/ オフ（0%）トグル |
| `ESC` | ゲーム終了 |
